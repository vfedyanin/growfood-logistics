## 3. Операционные сценарии

### TC-3.1: Создание рейса
**Предусловие:** Справочники заполнены

| Шаг | Действие | Ожидаемый результат |
|-----|----------|---------------------|
| 1 | Открыть `/operations/trips` | Список рейсов |
| 2 | Нажать "Создать рейс" | Форма создания |
| 3 | Заполнить: номер `TRP-2026-03-25-001` | Поле заполнено |
| 4 | Выбрать звено: Колпино → РЦ Москва | Автозаполнение origin/destination |
| 5 | Выбрать перевозчика: ТК Деловые Линии | Перевозчик выбран |
| 6 | Выбрать ТС: А123БВ777 | ТС выбрано |
| 7 | Указать плановые даты отправления/прибытия | Даты заполнены |
| 8 | Указать план: 20 паллет, 12000 кг | План заполнен |
| 9 | Сохранить | Рейс создан со статусом `PLANNED` |

### TC-3.2: Добавление грузовых единиц в рейс
| Шаг | Действие | Ожидаемый результат |
|-----|----------|---------------------|
| 1 | Открыть рейс `TRP-2026-03-25-001` | Детали рейса |
| 2 | Перейти на вкладку "Груз" | Список грузовых единиц (пустой) |
| 3 | Добавить паллету: barcode `PLT-001`, вертикаль GROWFOOD, 600 кг | Паллета добавлена |
| 4 | Добавить паллету: barcode `PLT-002`, вертикаль RETAIL, заказчик Магнит, 600 кг | Паллета добавлена |
| 5 | Добавить паллету: barcode `PLT-003`, вертикаль LAAS, заказчик ООО Ромашка, 600 кг | Паллета добавлена |
| 6 | Проверить сумму | Итого: 3 паллеты, 1800 кг |

### TC-3.3: Сборная паллета (mixed)
| Шаг | Действие | Ожидаемый результат |
|-----|----------|---------------------|
| 1 | Создать грузовую единицу: barcode `PLT-MIX-001`, тип `PALLET`, флаг `is_mixed = true` | Сборная паллета создана |
| 2 | Добавить состав: GROWFOOD 300 кг (50%), RETAIL/Магнит 300 кг (50%) | Состав добавлен |
| 3 | Добавить паллету в рейс | Паллета в рейсе |
| 4 | Проверить аллокацию стоимости после завершения рейса | Стоимость разделена 50/50 |

### TC-3.4: Завершение рейса с фактическими данными
| Шаг | Действие | Ожидаемый результат |
|-----|----------|---------------------|
| 1 | Открыть рейс `TRP-2026-03-25-001` | Статус: PLANNED |
| 2 | Изменить статус на `IN_TRANSIT` | Статус обновлён |
| 3 | Указать фактическое время отправления | Время записано |
| 4 | Изменить статус на `COMPLETED` | Статус обновлён |
| 5 | Указать фактическое время прибытия | Время записано |
| 6 | Указать фактические данные: 20 паллет, 11800 кг, стоимость 45000 ₽ | Факт записан |
| 7 | Проверить автоматическую аллокацию стоимости по грузовым единицам | Стоимость распределена пропорционально весу |

### TC-3.5: Событие качества (опоздание)
| Шаг | Действие | Ожидаемый результат |
|-----|----------|---------------------|
| 1 | Открыть завершённый рейс | Детали рейса |
| 2 | Если actual_arrival > planned_arrival + 30 мин | Система автоматически создаёт QualityEvent типа LATE_ARRIVAL |
| 3 | Или вручную создать событие: тип `LATE_ARRIVAL`, severity `MAJOR`, задержка 45 мин | Событие создано |

### TC-3.6: Событие качества (температура)
| Шаг | Действие | Ожидаемый результат |
|-----|----------|---------------------|
| 1 | Создать событие: тип `TEMP_VIOLATION` | Форма события |
| 2 | Указать: температура факт +8°C, требуемая -2...+4°C, длительность 30 мин | Событие создано |
| 3 | Severity автоматически = `CRITICAL` (выход за температурный режим) | Severity корректный |
